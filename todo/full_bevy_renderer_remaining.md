## File-size & cleanup remediation (RF) — per-file checklist

Per-file method (do in order; check the box ONLY after the file is ≤500 lines AND validated):
1. **Analyze** — map the file's exports, imports, and REAL consumers (`grep -rln`). Note whether it is on the runtime/boot path (boot-loaded) vs a leaf.
2. **Delete** — remove dead code (unreferenced functions/consts), redundant/duplicated logic, unused/unconnected code, leftover debug/probe/trace code, and obsolete examples/demos. For `examples/`/`lyrix` sub-apps: FIRST check whether the whole file is dead (no live import) → delete wholesale.
3. **Restructure** — extract cohesive owners into focused sibling modules so the file is ≤500 lines; reuse existing modules; no fallback, no shim, no duplicated ownership.
4. **Validate** — `node --check`; targeted tests; **boot probe** (`temp/boot_probe.mjs`) when the file is on the runtime/boot path; `npm run check:syntax`; relevant `check:*` guardrails.
5. **Maps** — update `maps/*` when ownership/API/structure changes.
6. **Check the box** with evidence: `before→after` line count + which validations passed.
