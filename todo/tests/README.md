# Registre des tests de non-régression

- `authentification.md` : compte, accès invité, téléphone, OTP, mot de passe et connexion.
- `main_menu.md` : barre d’outils principale, clics, appuis longs, routage pointeur et outils.
- `dashboard.md` : ouverture, fermeture, fluidité et interactions propres au Dashboard.
- `projets.md` : projets, objets, isolation, navigation, thumbnails et renommage.
- `preferences_utilisateur.md` : User Panel, préférences et fonds d’écran.
- `assistant_ai.md` : cycle de vie, voix, gestes, WebGPU, éjection, scènes et qualité visuelle de l’assistant.
- `transversal.md` : cohérence globale, rechargement et contrôles traversant plusieurs composants.

- [ ] Ajouter chaque nouveau test dans le fichier du composant qui possède la fonctionnalité.
- [ ] Ajouter un test de non-régression à chaque correction de bug avant de considérer la correction comme livrable.
- [ ] Conserver une seule action ou vérification par ligne de test.
- [ ] Ne supprimer un test que si la fonctionnalité correspondante est supprimée du produit.
